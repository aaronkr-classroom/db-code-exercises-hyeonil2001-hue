# 2장 연습문제

## 개념퍼즐

가로 )
3 복잡한 현실 세계의 데이터를 좀 더 쉽게 이해할 수 있도록 기술하는 개념적 도구: 데이터 모델

4 개념적 모델링 결과인 개념적 구조를 DBMS가 이해할 수 있는 논리적 구조, 즉 데이터 모델로 변환하는 과정: 논리적모델링

5 디스크에 데이터가 저장될 수 있도록 논리적 구조를 물리적 구조로 변환하는 과정: 물리적모델링

6 하나의 논리적 기능을 수행하기 위한 작업 단위: 트랜잭션

8 DB 설계 시 고려 사항 가운데 하나로, DB에 저장된 데이터 값은 제약조건을 항상 만족해야 한다는 것: 무결성

세로 )
1 현실 세계에서 인식한 데이터를 추상화를 통해 개념 세계의 추상적 개념으로 표현하는 과정: 개념적모델

2 현실 세계의 데이터를 DB에 표현하기 위한 모델을 만드는 과정: 데이터 모델링

7 DB 설계 시 고려 사항 가운데 하나로, 시스템 운영에 영향을 주지 않고 새로운 데이터나 응용 프로그램을 추가하는 것이 가능해야 한다는 것: 확장성

## 연습문제

개넘 세계의 데이터는 개체의 의미로부터 인식된 개념으로， 둘 이상의 속성으로 구성된 (______)으로표현된다.

3. 현실 세계의 직장인이 속한 세계에 따라 구분되는 구성 요소를 5개 이상 작성하시오.

|-----|----------|
| 현실 세계의 직장인 | 이름,나이,키,몸무게,성별,성격,연봉 | 
| 회사라는 개념 세계의 직장인 | 사원번호, 이름, 소속부서, 직급, 연봉 | 
| 자동차 보험사라는 개념 세계의 직장인 | 보험번호, 이름, 운전면허번호, 차종, 보험료 | 
| 회사라는 개념 세계와 대응하는 컴퓨터 세계의 직장인 | emp_id, name, dept, salary, position |

    c. 현실 세계에서 인석한 데이터를 추상화를 통해서 개념 세계의 추상적 개념으로 표현하는 과정

5. 데이터 모텔링을 개념적 모댈링， 논리적 모댈링， 물리적 모댈링으로 구분해서 수행히는 이유를 설명하시오.

6. 개념적 구조， 즉 개념적 모댈링 결과를 DBMS가 직접 이해할수 없는 이유를 설명하시오.
/ 개념적 모델링의 결과물은 er다이어그램으로 이는 사람이 이해하기 쉽게 하도록 만든 것이므로 컴퓨터가(dbms)가 이해하기 어려운 형태이다

8. 개념적 데이터 모벨의 구성 요소를 쓰시오.

9. 다음 중 데이터베이스를 구축하는 6단계를 바르게 나열한 것은? / 4
  1. 요구 수집 및 분석 - DBMS 선정 - 개념적 설계 - 논리적 설계 - 물리적 설계 - 구현 및 테스트
  2. 요구 수집 및 분석 - DBMS 선정 - 논리적 설계 - 개념적 설계 - 물리적 설계 - 구현 및 태스트
  3. 요구 수집 및 분석 - 개념적 설계 - DBMS 선정 - 논리적 설계 - 물리적 설계 - 구현 및 테스트
  4. 요구 수집 및 분석 - 개념적 설계 - 논리적 설계 - DBMS 선정 - 물리적 설계 - 구현 및 태스트

10. 다음 중 DB 설계 전략에 대한 설명이 잘못된 것은? / 3
  1. DB의 내용과 구조에 치중해서 설계히는 방법은 데이터 중심 설계다.
  2. 데이터의 처리와응용에 치중해서 설계하는 방법은 처리 중심 설계다.
  3. 데이터 중심 설계와 처리 중심 설계 가운데 하나를 선택해서 진행하는 것이 효율적이다.
  4. 데이터 중심 설계와 처리 중심 설계를 병행해서 진행히는 것이 효율적이다.

11. 다음 중 데이터베이스 설계 시 고려 사항에 대한 설명이 잘못된 것은? / 2
  1. 보안: 불법적인 접근(데이터의 변경 ， 손실， 노출 등)에 대해서 DB를 보호할 수 있어야한다.
  2. 확장성:시스템운영을일시중단하고새로운데이터나응용프로그램을 추가할수있어야한다.
  3. 일관성: 저장된 데이터 값간또는특정 질의에 대한응답들 간에 상호 모순 없이 일치해야한다.
  4. 무결성 :DB에 저장된 데이터 값은제약조건을항상만족해야한다.
